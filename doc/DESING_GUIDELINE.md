# Desing guidelines

---

## 🎯 1. デザインコンセプト

| 項目               | 詳細                                                                                                                                                                                        |
| :----------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **コアコンセプト** | **「Functional Minimalism for Experts」**<br>無駄な装飾を徹底的に排除し、コードと情報の高速な認知に特化した機能美を追求。シニアエンジニアが求める「正確性」「効率性」「硬派さ」を表現する。 |
| **ムード**         | 洗練、プロフェッショナル、クリーン、高速、硬派。                                                                                                                                            |
| **表現方法**       | モノトーン主体のフラットデザイン。ボーダーと余白でセクションを区切り、色彩は意味のあるアクセントに限定する。                                                                                |

## 🎨 2. スタイルと配色

### A. 配色（Color Palette）

| 項目                 | カラーコード (例)            | 用途                                                                 |
| :------------------- | :--------------------------- | :------------------------------------------------------------------- |
| **背景（ベース）**   | `#F9F9F9` (オフホワイト)     | ページ背景、サイドバー背景。                                         |
| **メイン（文字色）** | `#1A1A1A` (濃いダークグレー) | メインの文字色。コントラストを保ち、視認性を高める。                 |
| **サブテキスト**     | `#888888` (ミディアムグレー) | 投稿日、キャプションなど。                                           |
| **アクセント**       | `#007AFF` (Tech Blue)        | リンク、ボタン、タグの枠、検索時のハイライトなど、**唯一の色要素**。 |
| **コード背景**       | `#F5F5F5` (ライトグレー)     | インラインコード、コードブロックの背景色。                           |

### B. タイポグラフィ（フォント）

| 項目                | フォント指定                                                         | 特徴                                                                         |
| :------------------ | :------------------------------------------------------------------- | :--------------------------------------------------------------------------- |
| **見出し/本文**     | `system-ui, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` | OSネイティブフォントを優先。和文フォントの指定も同様にシステムフォント優先。 |
| **コード/技術用語** | `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace`   | 開発者にとって視認性の高い等幅フォントを優先指定。                           |

## 📐 3. レイアウトと構造

### A. 共通ヘッダー（グローバルナビゲーション）

| 要素               | PC版要件                                                                             | SP版要件                                                               |
| :----------------- | :----------------------------------------------------------------------------------- | :--------------------------------------------------------------------- |
| **ロゴ**           | 左寄せで「`</> Tech Blog`」を配置。ロゴアイコンは**`</>`**を使用。                   | 左寄せ。タイトルテキストは省略可。                                     |
| **検索**           | ヘッダー直下に常に表示。**Ctrl/Cmd + K**での全画面検索モーダル起動を強く示唆するUI。 | ハンバーガーメニュー内、または画面下部ナビゲーションにアイコンを配置。 |
| **ナビゲーション** | 右寄せで配置（記事一覧、タグ、概要）。                                               | ハンバーガーメニューに収納。                                           |
| **デザイン機能**   | ダークモード切り替えトグルはナビゲーションエリアに常に表示。                         | ハンバーガーメニュー内に配置。                                         |

### B. 記事一覧・検索結果

| 要素         | 要件                                                                                                            |
| :----------- | :-------------------------------------------------------------------------------------------------------------- |
| **デザイン** | ボーダーライン（1px）と余白で記事を区切るフラットなカードデザイン。                                             |
| **情報密度** | 1記事につき「**タイトル** (H3)」「**公開/更新日**」「**タグ**」「**サマリーテキスト** (2行まで)」を明確に表示。 |
| **画像**     | サムネイル画像は原則使用せず、テキスト情報を優先する。                                                          |

### C. 記事詳細ページ

| 要素            | 要件                                                                                    |
| :-------------- | :-------------------------------------------------------------------------------------- |
| **カラム構成**  | PC版は**2カラム**（メインコンテンツ + 右側に目次/TOC）。                                |
| **本文幅**      | 最大幅を1000px程度に制限し、適度な行長（60〜75文字）を確保。                            |
| **目次（TOC）** | PC版は右側のサイドバーに**追従型**で常に表示。現在地がハイライトされる。                |
| **サイドバー**  | 目次の下に**検索ウィジェット**と**人気タグ**のみを配置。                                |
| **スマホ**      | 1カラム表示。目次は記事タイトル直下のアコーディオン、またはページ最下部のボタンで表示。 |

## 🔧 4. 記事内要素（Markdownスタイル）

| 要素                  | 要件                                                                                                         |
| :-------------------- | :----------------------------------------------------------------------------------------------------------- |
| **見出し (H2, H3)**   | H2は本文より一回り大きく、下部に1pxのアクセントカラーのボーダー。H3はサイズと太字のみで区別。                |
| **コードブロック**    | **Nuxt Contentのシンタックスハイライト機能**を最大限活用。行番号を常に表示。モバイルでの横スクロールは必須。 |
| **引用 (Blockquote)** | 本文より文字サイズをやや小さくし、左側に縦のアクセントカラーライン（3px程度）を入れる。                      |
| **リンク**            | アクセントカラー（Tech Blue）で表示し、ホバー時にアンダーラインを付与。                                      |

## 🚀 5. 機能性とUX要件

| 項目               | 要件                                                                                           |
| :----------------- | :--------------------------------------------------------------------------------------------- |
| **検索UX**         | **検索性を最優先**。Ctrl/Cmd + Kで検索モーダルが即座に起動する機能を実装。                     |
| **スマホ可読性**   | `line-height`を**1.8以上**に設定し、スマートフォンでの目の疲れを軽減。                         |
| **タグ機能**       | サイト内の**全タグ**を一覧できる専用ページを設ける。各タグでのフィルタリング機能を必須とする。 |
| **パフォーマンス** | 全体の表示速度を最優先（Core Web Vitalsの高い水準での達成）。                                  |
